# UI 결합 원칙

> UI는 채팅 앱처럼 보이면 안 된다. 대화 UI가 공간을 덮어버리지 않는다.

## 1. 원칙: 종이 한 장

- 화면 위 UI는 **원고지 한 장**(`#FAF8F4`)뿐이다. 말풍선·아바타·타임스탬프·전송 아이콘·
  읽음 표시 등 채팅 문법 전면 금지.
- 종이는 공간 "위에 놓인 사물"이다 — 떠 있는 패널이 아니라, 그 자리에서 쓰고 있는 원고.

## 2. 배치와 크기

| 항목 | 데스크톱 | 모바일 |
|---|---|---|
| 종이 폭 | 최대 36rem, 화면의 55% 이하 | 화면 폭 - 2.5rem |
| 종이 위치 | 좌석의 시각 초점 반대편. A석=우측, B석=중앙 하단, C석=좌측 | 화면 하단 원고, 상단 32vh는 공간 유지 |
| 초점부 보호 | 벽난로 개구·북창·스탠드 갓은 종이에 가려지지 않는다 | 동일 |
| 입력창 | 종이의 마지막 줄 (본문과 같은 서체, 밑줄 하나) | 동일 |

## 3. 투명도와 분리

- 종이 불투명도 96%. 배경 블러는 종이 뒤 8px만 — 화면 전체 블러 금지.
- 어두운 장면에서는 색을 바꾸지 않고 그림자(`0 22px 60px rgba(0,0,0,0.35)`)로만 분리.
- 글자: 본문 17px/1.9, 사용자 필기 이탤릭, 최소 16px (모바일 확대 방지).

## 4. 공간과 UI의 우선순위

1. 안전·오류 알림 (드묾, 종이 위 문장으로)
2. 공간의 초점부 (불·창·스탠드)
3. 원고 종이
4. 그 외 모든 것 — 존재하지 않아야 정상

## 5. 상태별 규칙

- **문 앞: 입장 문장 확인.** 좌석 선택과 문장 발급은 사전 예약 단계에서 이미 끝나
  있다 (`MANRI_LIBRARY_EXPRESSION_V2.md` §0·§1). 문 앞에서 "예약하고 오셨나요?" —
  사용자가 발급받은 입장 문장을 건네면 확인되고, 예약된 자리로의 이동이 시작된다.
  둘러보기 모드에서는 이 단계 자체가 없다 (세션 없음).
- **좌석 이동 중: UI 전부 소멸.** 카메라 동선 동안 화면에는 공간만 있다.
  이동 종료 + 0.6s 후 원고가 아래에서 2cm 떠오르며 등장 (1.2s ease).
- 착석 직후: **잠시 머무름** — 공간만 있는 1.5s를 지나, 첫 물음 한 줄이 나타나고,
  0.8s 후 입력줄이 나타난다.
- 대화 중: 공간은 배경 밝기 변동 ±4% 이내 (조명 규칙 상속). UI는 스크롤 외 움직임 없음.
- 일어나기: 원고가 접히듯 사라진 뒤(0.9s) 마무리 종이가 새로 놓인다.
- 로딩: 서재의 문장("책장을 한 장 넘기고 있습니다" 계열)만. 스피너·점 애니메이션 금지.

## 6. 모바일 대응

- 상단 32vh: 좌석 장면 고정 (스크롤에도 유지) — 공간이 항상 화면에 남는다.
- 하단: 원고 스크롤 영역. 키보드가 올라오면 장면은 20vh로 줄되 사라지지 않는다.
- 가로 모드: 장면 좌측 40% / 원고 우측 60%.
