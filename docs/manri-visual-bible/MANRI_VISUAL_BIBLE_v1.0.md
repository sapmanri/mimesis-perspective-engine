# MANRI LIBRARY VISUAL BIBLE v1.0

- 상태: 검수 대기 (웹 구현·이미지 생성·좌석 이미지 확정은 검수 후)
- 원칙: 예쁘기보다 일관성. 화려함보다 머무를 수 있는 분위기. 모호한 표현보다 재현 가능한 수치.
- 목적: 어떤 생성 도구가 100번 만들어도 같은 서재로 유지되는 시각 기준.

> 만리서재는 AI 채팅 화면이 아니다. 사람이 질문을 들고 들어와, 자리를 옮기고, 속도를 늦추고,
> 질문을 통해 스스로 자신의 프레임을 바꾸는 디지털 공간이다.
> 공간은 장식이 아니다. 공간 자체가 사용자의 집중, 호흡, 감정, 사고의 속도를 바꾸는 장치다.

## 핵심 결정 (전 문서가 상속한다)

1. **하나의 방.** 세 좌석은 방 하나(7.2m × 5.4m, 천장 3.1m)에 함께 있다. 어느 자리에서도
   다른 두 자리가 시야에 걸린다. 분리감은 벽이 아니라 **빛의 웅덩이**로 만든다.
2. **방위가 빛의 법칙이다.** 입구=남, 벽난로=서, 책상=북측 창 아래, 독서 의자=북동 구석.
   북측 창은 직사광이 들지 않는 확산광 — 어떤 시각에도 책상 빛이 안정적인 이유다.
3. **불변 3종.** 가구 배치·건축 구조·오브젝트 12종(대장 참조)은 어떤 장면에서도 바뀌지 않는다.
   변하는 것은 빛, 창밖, 불의 세기, 공기의 밀도, 사물의 미세한 어긋남뿐이다.
4. **카메라는 사람이다.** 35mm 화각, 선 키 150cm / 앉은 키 115cm, 걸음 속도 0.7m/s.
   드론·광고·게임 트레일러 문법 전면 금지.
5. **공간은 연기하지 않는다.** 움직임은 낮은 빈도의 환경 미동뿐. 사용자가 생각하는 동안
   공간이 감정을 대신 표현하지 않는다.
6. **침묵이 기본값.** 음악 없음. 소리는 자리별 낮은 환경음이며 끌 수 있다.
7. **UI는 원고지 한 장.** 공간 위에 종이 한 장이 놓일 뿐, 채팅 앱의 어떤 문법도 쓰지 않는다.

## 완료 조건 자답

- **세 좌석은 같은 서재 안에 있는가?** 그렇다 — 단일 평면 위 좌표로 고정했고, 좌석 간
  시선 연결(각 좌석에서 보이는 다른 좌석의 실루엣)을 `spatial-layout.md` 6절에 명시했다.
- **어느 장면을 생성해도 가구 위치가 유지되는가?** 평면 좌표 + 프롬프트의 layout lock +
  오브젝트 대장의 위치 필드가 삼중으로 고정한다.
- **좌석별 역할이 비주얼만으로 구분되는가?** A=불의 주황(1900K)·낮은 카메라·흔들의자,
  B=북창의 회백(5000K)·정면 구도·종이와 금속, C=남빛 어스름(2400K 스탠드)·깊은 심도·직물.
  색온도·카메라·재질 축이 서로 겹치지 않는다.
- **낮과 밤이 달라도 같은 공간으로 보이는가?** 구조·가구·오브젝트 불변 + 시간 변화는
  `motion-and-sound.md` 3절의 허용 목록(빛·창밖·불·공기)로 제한된다.
- **생성 도구를 바꿔도 재현되는가?** 프롬프트가 자연어 묘사가 아니라 좌표·재질·잠금의
  계층 구조라서 도구 독립적이다. 모든 프롬프트에 5종 잠금(continuity/negative/camera/
  material/layout)을 동봉한다.
- **UI가 몰입을 깨지 않는가?** `ui-integration.md` — 종이 한 장 원칙, 이동 중 UI 소멸,
  초점부(벽난로·창) 가림 금지, 채팅 문법 금지.
- **첫 방문과 반복 방문 모두 견디는가?** 첫 방문=입장 동선 전체, 재방문=문 생략하고
  전실 프레임부터. 계절·날씨 변주가 반복 방문의 미세한 새로움을 담당한다
  (`lighting-and-camera.md` 카메라 문법 11절).
