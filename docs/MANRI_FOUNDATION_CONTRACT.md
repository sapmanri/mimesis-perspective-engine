# MANRI FOUNDATION CONTRACT v1.0

- 성격: Perspective Engine(Foundation)과 만리서재 경험층 사이의 **상호 무지(無知) 계약**
- 상태: 검수 대기 (Vase 발안 2026-07-19)
- 위계: `MIMESIS_CONSTITUTION.md` 아래, `MANRI_INTEGRATION_BIBLE.md`의 부속 계약.
  Integration Bible이 "두 층을 어떻게 **연결**하는가"라면, 이 문서는
  "두 층이 무엇을 서로 **몰라야** 하는가"다.

## 왜 무지가 계약인가

두 층이 서로를 알수록 시스템은 한 덩어리가 되어 간다. 엔진이 좌석을 알면 좌석의
사정이 엔진 규칙에 스며들고, 대화 문서가 엔진 내부를 복사하면 같은 규칙을 두 곳에서
고쳐야 한다. 10년 뒤에도 모델과 서비스를 갈아끼우려면, 연결만큼 **단절**이 명세되어야
한다. 이 계약은 그 단절의 명세다.

## 제1조 — 엔진이 절대 알지 못하는 것

Perspective Engine(Genome Edition 1 · Engine Spec v1 · Blueprint v1)은 다음 다섯을
알지 못한다. 엔진 문서와 엔진의 산출 정의 어디에든 이 개념이 등장하면 계약 위반이다.

1. **공감** — 감정을 받는 방식은 경험층의 일이다 (`conversation-philosophy.md`).
   엔진은 감정을 관찰된 Feature로만 다룬다.
2. **UI** — 화면·종이·말풍선·원고의 존재를 엔진은 모른다.
3. **말투** — 말끝·호흡·온도의 구체는 `language-style.md`가 정한다.
   엔진이 갖는 것은 원리(E-Rules)까지다 — 그 대화체 발현은 경험층 소관.
4. **예약** — 예약 문장·입장의식·좌석 선택의 존재를 엔진은 모른다.
5. **공간** — 서재·벽난로·창·빛·소리를 엔진은 모른다. 좌석 맥락은 파이프라인
   ④~⑤(경험층 단계)에서만 적용되며, ②(엔진 계산)에는 절대 입력되지 않는다.

검증 기록 2026-07-19: 엔진 3문서 전문 검색에서 좌석·벽난로·예약·공감·말풍선·원고지·
서재·UI 출현 0회 — 현재 상태 계약 준수.

## 제2조 — 경험층이 절대 하지 않는 것

1. Conversation Bible · Visual Bible은 엔진의 내부 Rule을 **복사·변형·재구현하지
   않는다.** 참조만 한다 — Rule 번호로 가리킨다 (예: "Rule E-02", 본문 인용 금지).
2. 경험층의 필요 때문에 Genome·Lexicon·Rule을 수정하지 않는다. 필요가 생기면 그것은
   "새 형질/Rule의 발견"이며 설계자 판정 절차를 탄다. (Integration Bible §5의 승격)
3. 경험층은 엔진의 계산을 대신하지 않는다 — 관점 이동의 *후보*는 언제나 엔진의
   산출이고, 경험층은 그중의 *걸음*(또는 T0 — 걸음 없음)만 정한다.

## 제3조 — 접촉면은 하나다

두 층이 만나는 유일한 통로는 `MANRI_INTEGRATION_BIBLE.md`의 파이프라인(①→⑥)이다.
새 접촉면이 필요해지면 이 계약과 Integration Bible의 개정으로만 만든다 — 코드나
프롬프트에서 임의로 연결하지 않는다.

## 제4조 — 용어 규율

경험층 문서 안에서 Perspective Engine은 **Foundation**이라 부른다 (내부 호칭).
"엔진"이라는 낱말을 파이프라인 전체(대화 시스템)의 뜻으로 쓰지 않는다 — Foundation과
파이프라인의 혼용은 이 계약의 경계를 흐린다. (현행 Expression V2 §1 "착석한 자리는
엔진에 맥락으로 전달"은 이 규율 위반 사례 — v2.1 개정 대상. 의미는 "파이프라인의
표현 단계에 전달"이다.)

## 제5조 — 위반 판정 (검수 체크리스트)

- [ ] 엔진 문서 또는 엔진 산출 정의에 제1조의 다섯 개념이 등장하는가
- [ ] 경험층 문서에 엔진 Rule의 본문이 복사·변형되어 있는가 (번호 참조는 허용)
- [ ] Integration Bible 밖에서 두 층을 직접 연결하는 서술·코드가 있는가
- [ ] "엔진"이 Foundation 아닌 것을 가리키는 문장이 있는가

하나라도 '그렇다'면 해당 문서·코드는 확정될 수 없다.
